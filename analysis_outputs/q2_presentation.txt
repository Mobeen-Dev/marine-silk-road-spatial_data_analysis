DATA QUALITY ISSUES FOUND
================================================================================

Issue 1: File Size vs. Memory
-----------------------------
Problem: Each raster needs 9.12 GB RAM to load
Impact: Cannot load entire file at once
Solution: ✓ SOLVED - Implemented chunked reading (1000×1000 pixels)

Code proof: data_analysis_chunked.py successfully processed all 4 files


Issue 2: Extreme Value Range
----------------------------
Problem: Values range from 0 to 65 million (huge spread)
Impact: Direct visualization produces poor contrast
Example: On a 0-255 color scale, everything appears black except ports

Evidence:
• 91.70% = 0 (black)
• 6% = 1-10 (nearly black)
• 2% = major shipping (visible)

Solution: REQUIRES log transformation or percentile clipping


Issue 3: Coordinate System Distortion
-------------------------------------
Problem: WGS84 (lat/lon) causes distortion at high latitudes
Impact: Grid cells are smaller near poles

Cell size comparison:
• Equator: 556m × 556m = 0.31 km²
• 60°N: 556m × 278m = 0.15 km² (50% smaller!)
• 80°N: 556m × 97m = 0.05 km² (85% smaller!)

This means density values aren't directly comparable across latitudes.

Solution: Reproject to equal-area for analysis, keep WGS84 for storage


Issue 4: 90% Wasted Storage
---------------------------
Problem: 90% of pixels store zeros (no activity)
Impact: Inefficient processing, large file sizes

Opportunity: Convert to sparse representation
• Current: 2.45 billion pixels stored
• Optimized: ~245 million non-zero pixels (10%)
• Potential: 90% size reduction

Solution: Create filtered versions for hotspot analysis


Issue 5: No Seasonal Information
--------------------------------
Problem: 6 years aggregated into single cumulative value
Impact: Cannot see seasonal variations

Examples we're missing:
• Summer cruise season spikes
• Winter ice route changes
• Pandemic impact (2020-2021)

Note: This is data limitation, not quality issue


================================================================================
REQUIRED PREPROCESSING STEPS
================================================================================

BEFORE VISUALIZATION:
--------------------

Step 1: Remove NoData Values ⭐ CRITICAL
What: Filter out invalid pixels (value = 2,147,483,647)
Why: Prevents corruption in statistics
Code:
```python
valid_data = data[data != 2147483647]
```


Step 2: Log Transformation ⭐ CRITICAL  
What: Apply log10(value + 1) transformation
Why: Compresses 0-65M range to 0-7.8 range
Result: Low-density shipping lanes become visible

Example transformation:
• 0 → 0 (still zero)
• 10 → 1.04
• 1,000 → 3.00
• 1,000,000 → 6.00
• 65,000,000 → 7.81

Before: 99% of values invisible on color scale
After: Full range of shipping activity visible


Step 3: Percentile Clipping ⭐ HIGH PRIORITY
What: Clip to 99th percentile (remove top 1%)
Why: Top 1% (major ports) dominate entire color scale
Result: Better contrast for shipping lanes

Technique:
```python
upper_limit = np.percentile(data, 99)
clipped_data = np.clip(data, 0, upper_limit)
```


Step 4: Spatial Downsampling ⚠ RECOMMENDED
What: Reduce resolution for global maps
Why: 72K × 34K pixels too detailed for overview
Options:
• 100x downsample → 720 × 340 pixels (global view)
• 50x downsample → 1440 × 680 pixels (regional)
• 10x downsample → 7200 × 3400 pixels (detailed)

Use average aggregation to preserve density information.


Step 5: Separate Land from Ocean ⚠ RECOMMENDED
What: Distinguish land-zeros from ocean-zeros
Why: Better visual clarity
Approach:
• Land zeros: Gray/brown background
• Ocean zeros: Light blue or transparent
• Shipping: Color gradient (yellow-orange-red)


BEFORE HOTSPOT DETECTION:
-------------------------

Step 6: Apply Threshold Filter
What: Keep only cells with significant traffic
Why: Focus on actual hotspots, ignore noise
Suggested threshold: >1,000 AIS positions
Result: ~1% of pixels remain


Step 7: Coordinate Transformation
What: Reproject to equal-area projection (e.g., Mollweide)
Why: Accurate distance and area calculations
When: Before clustering or distance-based algorithms


BEFORE CATEGORY COMPARISON:
---------------------------

Step 8: Normalization
What: Scale each category to 0-1 range
Why: Enable fair comparison
Need: Commercial max = 62M, Passenger max = 112K
Without normalization, Commercial dominates every comparison.

Code:
```python
normalized = data / data.max()
```
