Issue 3: COORDINATE SYSTEM CONSIDERATIONS
------------------------------------------
Problem: EPSG:4326 (unprojected) causes distortion in high latitudes
Impact: 
- Grid cells near poles are smaller in actual area
- Distance calculations inaccurate
- Density comparisons latitude-dependent

Evidence:
Cell size at equator: ~556m × 556m = 0.31 km²
Cell size at 60°N: ~556m × 278m = 0.15 km²
Cell size at 80°N: ~556m × 97m = 0.05 km²

Visualization Impact:
- Polar regions appear "stretched" east-west
- Density values not directly comparable across latitudes

Solutions:
1. Reproject to equal-area projection (e.g., Mollweide, Robinson)
2. Apply latitude correction factor to density values
3. Limit analysis to ±60° latitude
4. Use web Mercator (EPSG:3857) for interactive maps only

Recommendation: Keep WGS84 for storage, reproject on-the-fly for display

Status: ⚠ REQUIRES PROJECTION STRATEGY

def log_transform(data, base=10):
    """Log transform for visualization"""
    # Add 1 to handle zeros (log(0) = undefined)
    return np.log(data + 1) / np.log(base)

STEP 3: PERCENTILE CLIPPING (RECOMMENDED)
------------------------------------------
Objective: Improve contrast by removing outliers
Method:
```python
def percentile_clip(data, lower=0, upper=99):
    """Clip to percentile range"""
    p_low = np.percentile(data[data > 0], lower)
    p_high = np.percentile(data[data > 0], upper)
    return np.clip(data, p_low, p_high)

# Clip to 99th percentile (removes top 1% extremes)
clipped = percentile_clip(data, upper=99)
```

Justification:
- Top 1% values (major ports) dominate color scale
- Clipping reveals mid-density patterns (shipping lanes)
- Adjustable for different visualization goals

Priority: HIGH (improves visualization quality)
Computational Cost: O(n log n) - percentile calculation


STEP 4: SPATIAL RESAMPLING (CONDITIONAL)
-----------------------------------------
Objective: Reduce data size for faster processing/visualization
Method: Downsample using aggregation

```python
def resample_raster(src, scale_factor):
    """Downsample raster by scale factor"""
    new_height = src.height // scale_factor
    new_width = src.width // scale_factor
    
    data = src.read(
        1,
        out_shape=(new_height, new_width),
        resampling=Resampling.average  # or 'max' for hotspots
    )
    return data

# Example: Reduce from 72K×34K to 7.2K×3.4K (100x fewer pixels)
downsampled = resample_raster(src, scale_factor=10)
```

Resampling Methods:
- AVERAGE: Best for density visualization (preserves intensity)
- MAX: Best for hotspot detection (preserves peaks)
- MIN: Not recommended (loses information)
- SUM: For total traffic calculations

When to Apply:
- Global overview maps: 100x downsample (720×340 pixels)
- Regional maps: 10x downsample (7200×3400 pixels)
- Detailed analysis: No resampling (full resolution)

Priority: OPTIONAL (performance optimization)
Computational Cost: O(n/k²) where k = scale_factor

2.2 PREPROCESSING PIPELINE DESIGN
----------------------------------

RECOMMENDED WORKFLOW:
```python
def preprocess_marine_traffic_raster(raster_path, config):
    """
    Complete preprocessing pipeline
    
    Args:
        raster_path: Path to GeoTIFF file
        config: dict with preprocessing parameters
    
    Returns:
        Preprocessed data ready for visualization/analysis
    """
    
    with rasterio.open(raster_path) as src:
        # STEP 1: Load with optional resampling
        if config.get('downsample_factor'):
            data = src.read(
                1,
                out_shape=(
                    src.height // config['downsample_factor'],
                    src.width // config['downsample_factor']
                ),
                resampling=Resampling.average
            )
        else:
            data = src.read(1)
        
        # STEP 2: Filter NoData
        nodata = src.nodata
        data = np.where(data == nodata, np.nan, data)
        
        # STEP 3: Apply threshold (remove very low traffic)
        if config.get('min_threshold'):
            data = np.where(data < config['min_threshold'], 0, data)
        
        # STEP 4: Transform values
        if config.get('transform') == 'log':
            data = np.log10(data + 1)
        elif config.get('transform') == 'sqrt':
            data = np.sqrt(data)
        
        # STEP 5: Percentile clip
        if config.get('percentile_clip'):
            valid = data[~np.isnan(data) & (data > 0)]
            upper = np.percentile(valid, config['percentile_clip'])
            data = np.clip(data, 0, upper)
        
        # STEP 6: Normalize if requested
        if config.get('normalize'):
            max_val = np.nanmax(data)
            data = data / max_val
        
        return data, src.transform, src.crs
```

def validate_preprocessed_data(data, original_stats):
