OBJECTIVE 1: HEATMAP GENERATION
================================================================================

Goal: Create continuous density visualizations showing traffic intensity

PRIMARY TECHNIQUE: Gaussian Smoothing
─────────────────────────────────────

Method: Direct raster visualization with Gaussian filter
Tool: scipy.ndimage.gaussian_filter + Folium/Matplotlib

Why This Works:
✓ Data is ALREADY density (no need to convert)
✓ Just need smoothing for visual appeal
✓ Fast: O(n) complexity
✓ Preserves data fidelity

Process:
```
Raw Raster → Log Transform → Gaussian Smooth → Color Map → Map Overlay
```

Example Code:
```python
from scipy.ndimage import gaussian_filter

# Smooth the density data
smoothed = gaussian_filter(log_density, sigma=2)

# sigma controls smoothness:
# sigma=1: Sharp (preserves detail)
# sigma=2: Light smoothing ← RECOMMENDED
# sigma=5: Heavy smoothing (overview)
```

Parameters:
• sigma = 2 for 500m resolution data
• Log10 transformation (compress 0-65M to 0-7.8)
• 99th percentile clipping (remove extreme outliers)

Output:
• Static maps: PNG/PDF via Matplotlib
• Interactive: HTML via Folium
• Processing time: 2-5 minutes per category


Color Mapping Strategy:
----------------------
Dark Blue → Cyan → Yellow → Orange → Red
(Low density)                    (High density)

Separate colors for:
• Land: Gray/brown
• No traffic ocean: Light blue/transparent
• Shipping activity: Heat gradient


Multi-Scale Approach:
--------------------
• Global view: 100x downsample + heavy smoothing (sigma=5)
• Regional: 10x downsample + light smoothing (sigma=2)
• Detail: Full resolution + minimal smoothing (sigma=1)


Visualization Tools:
-------------------
Tool         | Best For              | Output
------------ | --------------------- | -------
Folium       | Interactive web maps  | HTML ← PRIMARY
Matplotlib   | Publication quality   | PNG/PDF
Plotly       | Dashboards            | HTML
Kepler.gl    | Large-scale explore   | HTML


SECONDARY ALTERNATIVE: Kernel Density Estimation (KDE)
──────────────────────────────────────────────────────

When to Use: If combining raster with point data (e.g., add port locations)

Method: Convert raster to points → Apply Gaussian KDE

Why NOT Primary:
✗ Computationally expensive
✗ Requires point conversion (data already density)
✗ Slower than direct approach


DECISION: Use Gaussian smoothing (PRIMARY)
──────────────────────────────────────────

Justification:
• Data format matches technique perfectly
• Fast and scalable
• Simple implementation
• Proven results

OBJECTIVE 2: HOTSPOT DETECTION
================================================================================

Goal: Identify statistically significant high-density regions

PRIMARY TECHNIQUE: DBSCAN + Getis-Ord Gi*
─────────────────────────────────────────

Two-Stage Approach:

Stage 1: DBSCAN (Clustering)
----------------------------
Purpose: Group nearby high-density pixels into clusters

Algorithm: Density-Based Spatial Clustering
- eps = 0.05° (~5 km search radius)
- min_samples = 10 (minimum cluster size)
- threshold = 10,000 AIS positions (pre-filter)

What it does:
1. Finds all pixels with density >10,000
2. Groups nearby pixels into clusters
3. Identifies noise (isolated hot pixels)
4. Outputs cluster centroids and boundaries

Result: ~50-200 candidate hotspots per category


Stage 2: Getis-Ord Gi* (Statistical Significance)
------------------------------------------------
Purpose: Test if clusters are statistically significant

What it does:
• Calculates z-score for each cluster
• Compares to surrounding "neighborhood"
• Tests: "Is this significantly hotter than expected?"

Significance Levels:
• Gi* > 1.96 → 95% confidence (p < 0.05) ✓
• Gi* > 2.58 → 99% confidence (p < 0.01) ✓✓
• Gi* > 3.29 → 99.9% confidence (p < 0.001) ✓✓✓

Result: Statistically validated hotspots only


Combined Workflow:
-----------------
```
Raw Data → Threshold Filter (>10K) → DBSCAN Clusters 
       → Calculate Gi* → Keep only significant (Gi*>1.96)
       → Rank by (density × significance) → Top 20 hotspots
```

Example Code:
```python
from sklearn.cluster import DBSCAN

# Step 1: Extract high-density points
high_density_mask = data > 10000
y, x = np.where(high_density_mask)

# Step 2: Cluster
clustering = DBSCAN(eps=0.05, min_samples=10)
labels = clustering.fit_predict(points)

# Step 3: Test significance with Gi*
for cluster in clusters:
    gi_star_score = calculate_gi_star(cluster)
    if gi_star_score > 1.96:
        significant_hotspots.append(cluster)
```

Output Format:
-------------
For each hotspot:
• Cluster ID
• Centroid coordinates (lat, lon)
• Bounding box
• Total density (sum of AIS positions)
• Max density (peak value)
• Size (number of grid cells)
• Gi* score (significance)
• Rank (sorted by combined score)

OBJECTIVE 3: MARINE HIGHWAY EXTRACTION
================================================================================

Goal: Extract major shipping route corridors (linear features)

PRIMARY TECHNIQUE: Morphological Skeletonization + Ridge Detection
──────────────────────────────────────────────────────────────────

Multi-Step Pipeline:

Step 1: Skeletonization
----------------------
Purpose: Extract centerlines of shipping lanes

Method: Morphological thinning
```
Wide Corridor → Binary Threshold → Fill Gaps → Skeletonize → Centerline
```

What it does:
• Converts wide shipping lanes to single-pixel-wide lines
• Preserves topology (connectivity)
• Provides width estimation (medial axis distance)

Code:
```python
from skimage.morphology import skeletonize, medial_axis

# Threshold to binary
route_mask = density > 5000

# Fill gaps
from scipy.ndimage import binary_dilation
filled = binary_dilation(route_mask, iterations=3)

# Extract centerline
skeleton = skeletonize(filled)
```


Step 2: Ridge Detection (Validation)
------------------------------------
Purpose: Identify linear features using filter

Method: Frangi filter (tubular structure detector)
```python
from skimage.filters import frangi

ridges = frangi(log_density, sigmas=range(1,10,2))
```

Why Frangi:
✓ Designed for tube-like structures (perfect for shipping lanes)
✓ Multi-scale (detects both wide and narrow lanes)
✓ Robust to noise


Step 3: Combine & Vectorize
---------------------------
```
Skeleton ∩ Ridges → Binary Routes → Vectorize → LineStrings
```

Vectorization: Convert raster lines to vector geometries
Tool: OpenCV contour detection + Shapely LineString


Step 4: Classification
----------------------
Categorize routes by:

Traffic Volume:
• Primary: >100,000 avg positions (major trade routes)
• Secondary: 10,000-100,000 (regional)
• Tertiary: <10,000 (minor)

Length:
• Long-haul: >5,000 km (intercontinental)
• Regional: 500-5,000 km
• Short-haul: <500 km

Connectivity:
• Hub-to-hub: Connects major ports
• Feeder: Minor to major ports
• Coastal: Along coastlines


Expected Routes:
---------------
Major Intercontinental:
• Trans-Pacific (Asia ↔ North America)
• Trans-Atlantic (Europe ↔ Americas)
• Asia-Europe (via Suez Canal)

Regional:
• Intra-Asia (China-Japan-Korea-SE Asia)
• Mediterranean routes
• Caribbean-US East Coast
• Europe-West Africa

Chokepoints:
• Suez Canal corridor
• Panama Canal approach
• Strait of Malacca
• Strait of Gibraltar
• English Channel


Post-Processing:
---------------
• Remove short segments (<50 km)
• Simplify lines (Douglas-Peucker algorithm)
• Merge nearby parallel routes
• Label routes (based on endpoints)


Complete Workflow:
-----------------
```python
from skimage.morphology import skeletonize
from skimage.filters import frangi
import cv2
from shapely.geometry import LineString

# Step 1: Extract skeleton
skeleton, widths = extract_skeleton(density, threshold=5000)

# Step 2: Ridge detection
ridges = frangi(np.log10(density + 1))

# Step 3: Combine (intersection)
routes_binary = skeleton & (ridges > 0.5)

# Step 4: Vectorize
contours = cv2.findContours(routes_binary.astype(np.uint8))
route_lines = [LineString(c.squeeze()) for c in contours]

# Step 5: Classify
for route in route_lines:
    avg_traffic = calculate_avg_density(route, density)
    if avg_traffic > 100000:
        route.category = 'Primary'
    elif avg_traffic > 10000:
        route.category = 'Secondary'
```


SECONDARY ALTERNATIVE: Least-Cost Path Analysis
───────────────────────────────────────────────

Method: Calculate optimal routes between known ports

Requirements:
• World Port Index (port locations)
• Cost surface = inverted density (high traffic = low cost)

Algorithm:
```python
from skimage.graph import route_through_array

# Invert density to cost
cost = max_density - density

# Find path between two ports
path = route_through_array(cost, start_port, end_port)
```

Advantages:
✓ Follows actual traffic patterns
✓ Connects specific endpoints
✓ Avoids land masses

Disadvantages:
✗ Requires port database
✗ Computationally expensive
✗ Only finds port-to-port routes

Use Case: Validation, gap-filling


DECISION: Use Skeletonization + Ridge Detection (PRIMARY)
─────────────────────────────────────────────────────────

Justification:
• Extracts ALL routes (not just port-to-port)
• Fast morphological operations
• Provides width estimation
• Proven in vessel traffic studies

Processing time: 30-60 minutes for global dataset
